Integrating ROS 2 with Your Simulation

Having a simulated robot that you can control with ROS 2 is a powerful combination. The key to making this work smoothly is the integration between the ROS 2 ecosystem and the simulator. This is typically achieved through a ROS bridge and managed with ROS 2 launch files.

The Role of the ROS Bridge​

A ROS bridge is a component that translates messages between the simulator's internal communication system and the ROS 2 network.

  • In Gazebo, the ROS bridge is provided by a set of plugins (like the ones we saw in the previous sections) that directly publish and subscribe to ROS 2 topics.
  • In Unity and Isaac Sim, there is a dedicated ROS Bridge component that you add to your simulation scene to connect to your ROS 2 application.

This bridge is what allows your ROS 2 nodes (e.g., for navigation, perception, or control) to communicate with the simulated robot and its environment as if it were real hardware.

Orchestration with ROS 2 Launch Files​

A typical robotics application consists of many different nodes that need to be started in the correct order. A ROS 2 launch file is a Python script that allows you to define and launch a complex system of nodes with a single command.

When working with simulation, a launch file is used to:

  1. Start the simulator (e.g., Gazebo).
  2. Spawn your robot model into the simulation.
  3. Start the robot_state_publisher to publish the robot's state.
  4. Start RViz for visualization.
  5. Start any other nodes required for your application (e.g., navigation, control).

Visualizing with RViz​

RViz is a powerful 3D visualization tool for ROS. It allows you to view the state of your robot and its sensor data in a 3D environment. When working with simulation, you can use RViz to:

  • Display the 3D model of your robot.
  • Visualize sensor data, such as Lidar scans, camera images, and IMU orientation.
  • Monitor the output of your navigation stack, such as the planned path and costmaps.
  • Send commands to your robot, such as setting a navigation goal.

Example: A Complete Simulation Launch File​

Let's look at an example of a ROS 2 launch file that starts Gazebo, spawns a robot, and starts a robot_state_publisher and RViz.

import os
from ament_index_python.packages import get_package_share_directory
from launch import LaunchDescription
from launch.actions import IncludeLaunchDescription
from launch.launch_description_sources import PythonLaunchDescriptionSource
from launch_ros.actions import Node

def generate_launch_description():
    # Get the path to the Gazebo ROS launch file
    gazebo = IncludeLaunchDescription(
        PythonLaunchDescriptionSource([os.path.join(
            get_package_share_directory('gazebo_ros'), 'launch'), '/gazebo.launch.py']),
        )

    # Get the path to the URDF file
    urdf_file_name = 'my_robot.urdf'
    urdf = os.path.join(
        get_package_share_directory('my_robot_description'),
        'urdf',
        urdf_file_name)

    # Spawn the robot in Gazebo
    spawn_entity = Node(package='gazebo_ros', executable='spawn_entity.py',
                        arguments=['-topic', 'robot_description',
                                   '-entity', 'my_robot'],
                        output='screen')

    # Start the robot state publisher
    robot_state_publisher = Node(
        package='robot_state_publisher',
        executable='robot_state_publisher',
        name='robot_state_publisher',
        output='screen',
        parameters=[{'robot_description': open(urdf).read()}])

    # Start RViz
    rviz = Node(
        package='rviz2',
        executable='rviz2',
        name='rviz2',
        arguments=['-d', os.path.join(get_package_share_directory('my_robot_description'), 'rviz', 'default.rviz')])

    return LaunchDescription([
        gazebo,
        robot_state_publisher,
        spawn_entity,
        rviz,
    ])

This launch file does the following:

  1. Includes the standard gazebo.launch.py to start the Gazebo simulator.
  2. Finds the URDF file for the robot.
  3. Uses the spawn_entity.py script to spawn the robot in Gazebo.
  4. Starts a robot_state_publisher to publish the robot's state based on the URDF.
  5. Starts RViz with a pre-configured view.

By creating a launch file like this, you can start your entire simulation environment with a single command: ros2 launch my_robot_description my_simulation.launch.py

This tight integration between ROS 2 and simulation is what enables the "digital twin" development philosophy, where you can seamlessly switch between developing on a simulated robot and a physical one.
