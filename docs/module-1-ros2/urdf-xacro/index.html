Describing Your Robot with URDF and Xacro

To work with a robot in ROS 2, especially in a simulator, you need a way to describe its physical structure. This is where the Unified Robot Description Format (URDF) comes in.

URDF is an XML-based format for representing a robot model. It describes the robot's links, joints, sensors, and their relationships.

Why is URDF Important?​

A URDF file is a "digital twin" of your robot's physical structure. It's used by many tools in the ROS ecosystem:

  • Simulation: Simulators like Gazebo and Isaac Sim use URDF to create a 3D model of your robot in the virtual world.
  • Visualization: Tools like RViz2 use URDF to display a 3D model of your robot, which is useful for visualizing sensor data and debugging.
  • Kinematics and Dynamics: Libraries like the robot_state_publisher use URDF to compute the robot's kinematics (the position of each link) and dynamics.

The Core Components of URDF​

A URDF file is made up of two main components: <link> and <joint>.

<link>​

A link represents a rigid part of the robot, such as a wheel, a gripper, or a chassis. Each link has a name and can have the following properties:

  • <visual>: Describes the visual appearance of the link (shape, color, texture).
  • <collision>: Describes the collision geometry of the link, which is used by physics engines for collision detection.
  • <inertial>: Describes the inertial properties of the link (mass, center of gravity, inertia tensor).

<joint>​

A joint connects two links together and defines how they can move relative to each other. Each joint has a name, a parent link, a child link, and a type. Common joint types include:

  • revolute: A rotational joint with a single axis of rotation (e.g., a wheel).
  • continuous: A revolute joint with no angle limits.
  • prismatic: A translational joint that moves along a single axis.
  • fixed: A joint that does not allow any motion.

Simplifying URDF with Xacro​

URDF can become very verbose and repetitive for complex robots. Xacro (XML Macros) is a macro language that helps to simplify URDF files. With Xacro, you can:

  • Define constants: Create variables for commonly used values (e.g., wheel_radius).
  • Create macros: Define reusable snippets of XML, such as a macro for a wheel that can be instantiated multiple times.
  • Perform mathematical calculations: Use simple math expressions to calculate values.

Xacro files are processed to generate a final URDF file. It is a very common practice to write robot descriptions in Xacro and then convert them to URDF.

Example: A Simple Two-Wheeled Robot​

Here is a simplified example of a URDF file (written in Xacro) for a two-wheeled robot.

<?xml version="1.0"?>
<robot name="my_robot" xmlns:xacro="http://www.ros.org/wiki/xacro">

  <!-- Constants -->
  <xacro:property name="wheel_radius" value="0.1" />
  <xacro:property name="wheel_length" value="0.05" />
  <xacro:property name="chassis_size" value="0.5 0.3 0.1" />

  <!-- Base Link (Chassis) -->
  <link name="base_link">
    <visual>
      <geometry>
        <box size="${chassis_size}" />
      </geometry>
    </visual>
    <collision>
      <geometry>
        <box size="${chassis_size}" />
      </geometry>
    </collision>
  </link>

  <!-- Macro for a wheel -->
  <xacro:macro name="wheel" params="prefix x_offset y_offset">
    <link name="${prefix}_wheel_link">
      <visual>
        <geometry>
          <cylinder radius="${wheel_radius}" length="${wheel_length}" />
        </geometry>
      </visual>
      <collision>
        <geometry>
          <cylinder radius="${wheel_radius}" length="${wheel_length}" />
        </geometry>
      </collision>
    </link>

    <joint name="${prefix}_wheel_joint" type="continuous">
      <parent link="base_link"/>
      <child link="${prefix}_wheel_link"/>
      <origin xyz="${x_offset} ${y_offset} 0" rpy="1.5707 0 0" />
      <axis xyz="0 1 0"/>
    </joint>
  </xacro:macro>

  <!-- Instantiate the wheels -->
  <xacro:wheel prefix="left" x_offset="0.1" y_offset="0.175" />
  <xacro:wheel prefix="right" x_offset="0.1" y_offset="-0.175" />

</robot>

This example defines a base_link for the chassis and then uses a Xacro macro to create two wheels. The macro is instantiated twice, once for the left wheel and once for the right wheel, with different parameters.

This is a very simple example, but it illustrates the power of URDF and Xacro for creating clean, modular, and reusable robot descriptions.
