Glossary of Key Terms

This glossary provides definitions for important terms and concepts used throughout this book, covering ROS 2, NVIDIA Isaac, Physical AI, and general robotics and AI terminology.

A​

Action (ROS 2): A long-running, goal-oriented task in ROS 2 that provides continuous feedback and can be preempted.

Actuator: A component of a robot that is responsible for moving or controlling a mechanism or system (e.g., motors, hydraulic cylinders).

AI (Artificial Intelligence): The simulation of human intelligence processes by machines, especially computer systems.

AMCL (Adaptive Monte Carlo Localization): A probabilistic localization algorithm used in ROS 2 to estimate a robot's pose within a known map.

Autonomous Robot: A robot that can perform tasks without continuous human guidance.

C​

Capstone Project: A culminating project that integrates knowledge and skills acquired throughout a course of study.

colcon: A command-line tool used to build sets of software packages, particularly in the ROS 2 ecosystem.

CUDA: NVIDIA's parallel computing platform and programming model for GPUs.

cuDNN (CUDA Deep Neural Network library): A GPU-accelerated library of primitives for deep neural networks.

D​

DDS (Data Distribution Service): A middleware standard for real-time, publish-subscribe communication, used as the underlying transport layer in ROS 2.

Deep Learning: A subfield of machine learning that uses artificial neural networks with multiple layers (deep neural networks) to learn from data.

Digital Twin: A virtual replica of a physical object, process, or system that can be used for simulation, testing, and monitoring.

Docusaurus: An open-source static site generator used to build documentation websites.

Domain Randomization: A technique used in simulation to vary parameters of the environment (e.g., textures, lighting, object positions) to improve the generalization of trained models to the real world.

G​

Gazebo: An open-source 3D robotics simulator that accurately simulates robots, sensors, and environments.

GPU (Graphics Processing Unit): A specialized electronic circuit designed to rapidly manipulate and alter memory to accelerate the creation of images in a frame buffer intended for output to a display device. Essential for AI and robotics.

H​

Humanoid Robot: A robot designed to resemble the human body, typically with a torso, head, two arms, and two legs.

I​

IMU (Inertial Measurement Unit): An electronic device that measures and reports a body's specific force, angular rate, and sometimes the orientation of the body, using a combination of accelerometers, gyroscopes, and magnetometers.

Imitation Learning (IL): A machine learning paradigm where an agent learns a policy by observing demonstrations from an expert.

Isaac Gym: NVIDIA's platform for high-performance reinforcement learning, enabling massive parallel simulation on GPUs.

Isaac ROS: A collection of hardware-accelerated ROS 2 packages optimized for NVIDIA GPUs, providing modules for perception, navigation, and manipulation.

Isaac Sim: NVIDIA's robotics simulation platform built on Omniverse, offering photorealistic and physically accurate virtual environments.

Isaac SDK: NVIDIA's original software development kit for robotics, with many of its features now integrated into Isaac Sim and Isaac ROS.

J​

Joint (Robotics): A connection between two rigid bodies (links) in a robot, allowing for relative motion between them.

L​

Launch File (ROS 2): An XML or Python file used in ROS 2 to start and configure multiple nodes and other processes with a single command.

Lidar (Light Detection and Ranging): A remote sensing method that uses light in the form of a pulsed laser to measure ranges (variable distances) to the Earth. Used for mapping and obstacle detection.

Link (Robotics): A rigid body in a robot's kinematic chain.

LLM (Large Language Model): A type of artificial intelligence program that can recognize and generate text, translate languages, and answer questions in a conversational way.

Localization: The process of a robot determining its own position and orientation within a given map.

M​

Manipulation: The ability of a robot to interact with and move objects in its environment.

MDX (Markdown with JSX): An authoring format that lets you seamlessly write JSX in your Markdown files. Used by Docusaurus.

Mermaid: A JavaScript-based diagramming and charting tool that renders markdown-inspired text definitions to create diagrams dynamically.

MoveIt 2: A powerful open-source robotics manipulation platform for ROS 2, providing tools for motion planning, kinematics, and collision avoidance.

N​

Navigation: The process of a robot moving from one location to another while avoiding obstacles.

Nav2: The ROS 2 Navigation Stack, providing a complete solution for autonomous navigation of mobile robots.

NLU (Natural Language Understanding): A subfield of AI that focuses on enabling computers to understand the meaning and intent behind human language.

Node (ROS 2): The smallest executable unit of computation in ROS 2, performing a specific task.

O​

Object Detection: A computer vision technique that identifies and locates objects within an image or video.

Omniverse: NVIDIA's platform for connecting and building 3D applications and workflows.

P​

Parameter (ROS 2): Dynamic configuration values for ROS 2 nodes that can be changed at runtime.

Perception: The ability of a robot to sense and interpret its environment using sensors.

Physical AI: The integration of advanced AI models (especially large foundation models) with physical robotic bodies, enabling them to perceive, reason about, and act in the real world.

Pose Estimation: The process of determining the 3D position and orientation of an object or robot.

Publisher (ROS 2): A component within a ROS 2 node that sends messages to a specific topic.

R​

Reinforcement Learning (RL): A machine learning paradigm where an agent learns to make decisions by performing actions in an environment and receiving rewards or penalties.

Robot Operating System (ROS 2): A flexible framework for writing robot software, providing tools, libraries, and conventions for building complex robotics applications.

ros2_control: A ROS 2 framework for real-time control of robots, providing a standardized interface for hardware and software controllers.

RViz2: A 3D visualization tool for ROS 2 that allows you to view robot models, sensor data, and other information.

S​

Sensor: A device that detects and responds to some type of input from the physical environment (e.g., camera, Lidar, IMU).

Service (ROS 2): A synchronous request/response communication mechanism in ROS 2.

SLAM (Simultaneous Localization and Mapping): The process of a robot building a map of an unknown environment while simultaneously localizing itself within that map.

Subscriber (ROS 2): A component within a ROS 2 node that receives messages from a specific topic.

T​

TensorRT: NVIDIA's SDK for high-performance deep learning inference.

Topic (ROS 2): An asynchronous, one-to-many communication channel in ROS 2.

U​

Unity: A popular cross-platform game engine that is also used for high-fidelity robotics simulation.

URDF (Unified Robot Description Format): An XML format used in ROS to describe the kinematic and dynamic properties of a robot.

V​

VLA (Vision-Language-Action) Models: AI models that integrate visual perception, natural language understanding, and action generation to enable robots to perform tasks based on human commands.

X​

Xacro (XML Macros): A macro language that simplifies the creation of complex URDF files by allowing the use of variables and macros.
