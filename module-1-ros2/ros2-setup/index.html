Setting Up Your ROS 2 Environment

Getting your development environment set up correctly is the first and most crucial step to starting your journey with ROS 2. This guide will walk you through installing ROS 2, creating a workspace, and introducing you to the essential command-line tools.

Choosing a ROS 2 Distribution​

ROS 2 releases are called "distributions". Each distribution has a specific name (e.g., Humble, Iron, Rolling) and is supported for a certain period. For this book, we will be using ROS 2 Humble Hawksbill, which is a Long-Term Support (LTS) release, primarily targeting Ubuntu 22.04.

Installing ROS 2 on Ubuntu 22.04​

This is the officially supported and recommended platform for ROS 2 Humble.

The following script will install the full desktop version of ROS 2 Humble, which includes ROS, RViz, demos, tutorials, and more.

# Set up your sources
sudo apt update && sudo apt install curl gnupg lsb-release
sudo curl -sSL https://raw.githubusercontent.com/ros/rosdistro/master/ros.key -o /usr/share/keyrings/ros-archive-keyring.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/ros-archive-keyring.gpg] http://packages.ros.org/ros2/ubuntu $(lsb_release -cs) main" | sudo tee /etc/apt/sources.list.d/ros2.list > /dev/null

# Install ROS 2 packages
sudo apt update
sudo apt install ros-humble-desktop-full python3-colcon-common-extensions

# Source the setup file
echo "source /opt/ros/humble/setup.bash" >> ~/.bashrc
source ~/.bashrc

To check if your installation was successful, you can run the talker and listener demo nodes. In one terminal, run: ros2 run demo_nodes_cpp talker In another terminal, run: ros2 run demo_nodes_cpp listener You should see the listener printing messages from the talker.

Understanding the ROS 2 Workspace​

A ROS 2 workspace is a directory where you can create and manage your own ROS 2 packages. It's a good practice to work within a workspace to keep your projects organized.

Here's how to create a workspace:

mkdir -p ~/ros2_ws/src
cd ~/ros2_ws

The src directory is where you will place the source code for your ROS 2 packages.

To build your workspace, you use the colcon tool:

cd ~/ros2_ws
colcon build

colcon will build the packages in your src directory and install the output into the install directory.

After building, you need to source the workspace's setup file to make the packages available in your environment:

source ~/ros2_ws/install/setup.bash

It's a good idea to add this line to your ~/.bashrc file as well.

Essential Command-Line Tools​

ROS 2 provides a rich set of command-line tools for interacting with your ROS 2 system. Here are a few of the most common ones:

  • ros2 run <package_name> <executable_name>: Runs a node from a package.
  • ros2 topic list: Lists all the active topics.
  • ros2 topic echo <topic_name>: Prints the messages being published on a topic.
  • ros2 node list: Lists all the active nodes.
  • ros2 service list: Lists all the available services.
  • ros2 service call <service_name> <service_type> <arguments>: Calls a service.
  • ros2 param list: Lists the parameters of a node.

These tools are invaluable for debugging and understanding your ROS 2 system. Now that you have your environment set up, you're ready to start writing your first ROS 2 program!
